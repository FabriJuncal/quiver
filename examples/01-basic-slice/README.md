# Basic Slice Example

This example shows the full spec → slice → PR → evidence loop in a simple, stack-agnostic way.

## What to Read

1. `SPEC.md`
2. `slices/slice-01/slice.json`
3. `slices/slice-01/pr.md`
4. `EVIDENCE_REPORT.md`

## Scenario

A small team wants to document a first slice that introduces a user-facing onboarding flow. The example does not depend on any framework.

## How to Use This Example

1. Read the spec.
2. Inspect the slice JSON.
3. Read the PR note.
4. Compare the evidence report.

## Expected Outcome

After reading the example, a new contributor should understand how a slice is structured and how the documentation fits together.
